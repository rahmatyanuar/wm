Service Instances

This field...
Displays...
Current
The number of running services currently active.
Note:
This count reflects both top-level services and any child services they may call. This count does not include internal server services that are running from the wm.server. folder of the WmRoot package.

Uptime
Displays the length of time (hh:mm:ss) that the server has been running since it was last started.
Requests
The fields in this section display statistics about requests that this server is processing. A request represents a service request from an external client or via a trigger. Each request counts as 1 request even if it invokes multiple services. By default, requests for services included in system packages (those that begin with Wm) are not included. Use the watt.server.stats.packages.exclude parameter to change the list of package whose services are excluded from the requests statistics.

Memory
The fields in this section describe the availability of memory on the server. The graph shows the memory usage over the last 24 hours. The bars represent the average usage by hour as a percentage of total memory.
Maximum
Displays the maximum amount of memory the Java virtual machine (JVM) is allowed to acquire, as determined by the wrapper.java.maxmemory Java property specified in the custom_wrapper.conf file. Because the JVM reserves some survivor space for garbage collection, the displayed value depends on the underlying garbage collection algorithm used. Consequently, the displayed value might be less than the value set for the wrapper.java.maxmemory property.
Note:
For Microservices Runtime, the maximum memory is determined by the JAVA_MAX_MEM property in Integration Server_directory /bin/server.bat(sh).
Committed
Displays the amount of memory guaranteed to be available for use by the Java virtual machine.
Used
Displays the amount of memory currently used by the Java virtual machine.
Free
Displays the amount of unused memory that is currently available to the Java virtual machine.
