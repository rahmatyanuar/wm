Denial of Service Options
Use the Denial of Service Options page to limit the number of requests an Enterprise Gateway Server will accept and the number it will process concurrently. By limiting requests this way, you can prevent Denial of Service (DoS) attacks that flood Enterprise Gateway Server with requests in an attempt to interfere with server processing.
Note:
This page is disabled if webMethods API Gateway is installed on Integration Server.
To ensure that requests from trusted servers are not denied, you can configure a whitelist of IP addresses so that requests from these IP addresses are always allowed.
Important:
Enterprise Gateway runs on an Integration Server that has been configured as an Enterprise Gateway Server running in your DMZ. The Enterprise Gateway Server requires a webMethods Enterprise Gateway license.
Click Configure Global Denial of Service to configure global limits on requests. When global settings are in effect, Enterprise Gateway Server considers all requests, regardless of the IP address from which a request was sent.
Click Configure Denial of Service by IP Address to configure limits on requests based on the IP address from which a request was sent.
Global Denial of Service
Enabled
Whether Enterprise Gateway Server is enforcing limits on the total number of requests it will accept and the total number it will process concurrently, regardless of IP address from which a request was sent.
Maximum Requests
Maximum number of requests Enterprise Gateway Server will accept in the specified number of seconds.
Maximum Requests in Progress
Maximum number of requests Enterprise Gateway Server will run concurrently.
Block Interval
Length of time (in minutes) that Enterprise Gateway Server is to block client requests on every Enterprise Gateway external port.
Error Message
Custom message to send to the client, if desired, when a request is denied.
Trusted IP Address Range
The trusted IPv4 or IPv6 addresses so that requests from these IP addresses are always allowed. Use the classless inter-domain routing (CIDR) notation to specify a range of IP addresses. To specify an IP address range, enter the first IP address in the range followed by a forward slash (/) and a CIDR suffix.
Example IPv4 address range:
192.168.100.0/22 represents the IPv4 addresses from 192.168.100.0 to 192.168.103.255
148.20.57.0/30 represents the IPv4 addresses from 148.20.57.0 to 148.20.57.3
Example IPv6 address range:
f000::/1 represents the IPv6 addresses from f000:: to ffff:ffff:ffff:ffff:ffff:ffff:ffff:ffff.
2001:db8::/48 represents the IPv6 addresses from 2001:db8:0:0:0:0:0:0 to 2001:db8:0:ffff:ffff:ffff:ffff:ffff.
You can also specify a single IP address instead of an IP address range.
You can specify multiple IP addresses or IP address ranges separated by commas (,).
Denial of Service by IP Address
Enabled
Whether Enterprise Gateway Server is to enforce limits on the number of requests it will accept or the number of requests it will process concurrently from any individual IP address. For example, you can specify that, if Enterprise Gateway Server receives more than 15 requests from any individual IP address in 10 seconds, subsequent requests from that IP address will be denied.
Maximum Requests
Maximum number of requests Enterprise Gateway Server will accept from any individual IP address in the specified number of seconds.
Maximum Requests in Progress
Maximum number of requests Enterprise Gateway Server will run concurrently from the same IP address.
When Limit Exceeds
Action to take when the number of requests from a non-trusted IP address exceeds the limits specified in the Maximum Requests or Maximum Requests in Progress fields.

Add to Deny List
Enterprise Gateway Server will permanently add the IP address that exceeded the limit to the Enterprise Gateway deny list. With this setting in effect, the server blocks requests from the IP address at every Enterprise Gateway external port. The address remains in the deny list until an administrator deletes it from the list.
Block
Enterprise Gateway Server will temporarily add the IP address that exceeded the limit to the Enterprise Gateway deny list. With this setting in effect, the server blocks requests from the IP address at every Enterprise Gateway external port for a period of time that you specify.
Block Interval
Length of time (in minutes) that Enterprise Gateway Server is to block requests from an IP address when the number of requests exceeds the value specified in the Maximum Requests or Maximum Requests in Progress field. The server blocks client requests from this IP address at every Enterprise Gateway external port.
Error Message
Custom message to send to the client, if desired, when a request is denied.
Trusted IP Address Range
The trusted IPv4 or IPv6 addresses so that requests from these IP addresses are always allowed. Use the classless inter-domain routing (CIDR) notation to specify a range of IP addresses. To specify an IP range, enter the first IP address in the range followed by a forward slash (/) and a CIDR suffix.
Example IPv4 address range:
192.168.100.0/22 represents the IPv4 addresses from 192.168.100.0 to 192.168.103.255
148.20.57.0/30 represents the IPv4 addresses from 148.20.57.0 to 148.20.57.3
Example IPv6 address range:
f000::/1 represents the IPv6 addresses from f000:: to ffff:ffff:ffff:ffff:ffff:ffff:ffff:ffff.
2001:db8::/48 represents the IPv6 addresses from 2001:db8:0:0:0:0:0:0 to 2001:db8:0:ffff:ffff:ffff:ffff:ffff.
You can also specify a single IP address instead of an IP address range.
You can specify multiple IP addresses or IP address ranges separated by commas (,).
Denied IP Address List
IP Address
List of IP addresses that have been blocked or added to the Enterprise Gateway deny list for sending requests that exceed configured limits for the number of requests received or the number of requests to process concurrently. When the requested action is Block, an IP address is displayed in the list until it is automatically deleted after the block interval has passed. If the requested action is Add to Deny List, the IP address remains in the list until an administrator deletes it.
Important:
The Enterprise Gateway deny list takes precedence over the allow/deny list for the Enterprise Gateway external port.
Delete
Deletes the IP address from the deny list. After you delete the IP address, Enterprise Gateway Server will allow requests from this IP address again.
