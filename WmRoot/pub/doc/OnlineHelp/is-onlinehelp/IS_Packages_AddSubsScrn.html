Add subscriber
Use the Add Subscriber page to add a subscriber to a package that your server publishes. A subscriber is another webMethods Integration Server that your server publishes one or more packages to.
Important:
The subscribing server must be running when you add a subscriber. Your server will connect to it during this process.
Click Return to Publishing to return to the list of available releases.
Packages
Lists the names of packages on the webMethods Integration Server. To add a subscriber to a package, select the package from this list.
Host Name
Specifies the domain name or numeric IP address of the webMethods Integration Server that will subscribe to the package specified in Packages.
Host Port
Specifies the port number on which the subscribing server expects packages to be published. If you do not know what port this is, contact the administrator of the subscribing server.
Transport
Specifies the transport protocol that your server will use to publish this package to the subscribing server. Select HTTP or HTTPS from the list. Make sure the protocol you specify matches the type of listener that the subscriber has assigned to the port specified in Host Port.
Note:
When this server uses HTTPS to connect to the subscribing server to publish a package, it presents its default certificate (specified on the Certificates).
Remote User Name
Specifies the user name that your server will use to log on to the subscriber's webMethods Integration Server to publish this package. This user name must be a member of the Replicators ACL on the subscribing server. If you do not know this name, contact the administrator of the subscribing server.
Remote Password
Specifies the password for the user specified in Remote Password. If you do not know this password, contact the administrator of the subscribing server.
Notification Email
Optional. Specifies the e-mail address of the user who will be notified when you create a release of this package. (To use this option, you must also set the SMTP Server option on the Resources to specify the address of the SMTP server that your server will use to deliver e-mail messages.)
