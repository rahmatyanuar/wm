Account Locking Settings
Use the Account Locking Settings to view the account locking criteria for users. Integration Server locks a user account when a user fails to provide the correct password after a specified number of attempts.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Click Edit Account Locking Settings to change the account locking settings of users.
Click Locked Users to view the list of locked user accounts.
Account Locking Settings
Use this section to view the account locking settings for all users.
Enabled
Specifies whether account locking is enabled.
Maximum Login Attempts
Specifies the number of attempts in the specified time interval (minutes, hours, or days) to provide the correct password before Integration Server locks the account.
Lockout Duration
Specifies the duration for which the account remains locked.
Apply Account Locking Policy To
Specifies the list of users to whom the account locking settings apply. Default is All Users except Predefined Users.
