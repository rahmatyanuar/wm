Details for XID
:
Use the Details for XID page to view detailed information about the transaction, such as the resources involved, the state of the transaction on each resource, and the inferred status of the transaction. You can also use this screen to indicate whether the Integration Server should commit, roll back, forget, or do nothing about the transaction.
Click Return to XA Manual Recovery to return to the list of current resource settings.
Click Delete Transactions to delete the transaction from the XA recovery store. Delete the transaction if you do not want to resolve a transaction using Integration Server Administrator (for example, because you want to resolve the transaction by working with the resources directly).
Global 2PC State: Transaction State
Use this table to view detailed information for a specific XA transaction.
XA Resource
Fully qualified namespace name of the resource.
XA Exists?
Indicates whether the transaction's XID exists on the resource. Yes indicates the XID exists; No indicates that the XID does not exist. Unknown indicates that the Integration Server could not determine whether the XID exists on the resource.
State
Current state of the transaction on the resource. The values are the same as those in the Global 2PC State list. If the Integration Server cannot determine if the XID exists on the resource, this column displays the error message that occurred when the Integration Server contacted the resource.
Inferred Status
Assumed status of the transaction on the resource, based on the values of XID exists and State. Based on the possible combinations, statuses are as follows:

Desired Action
Select the action that you want the Integration Server to perform to resolve the transaction. This column lists the possible actions for each resource, based on the combination of the State and XID Exists? values for the resource. Select one of the following:
Select this action...
To...
Commit

Roll back
Roll back the transaction on the resource.
Forget
To erase the XID from the resource.
You might want to do this if the resource administrator heuristically committed or rolled back the transaction on the resource.
Do nothing
Do nothing to resolve the transaction.
You might want to do this if the resource administrator has already taken the correct action on the resource so you need take none, or the resource is down for an extended period.
Perform Action(s)
Performs the actions you selected for each transaction. If the actions complete successfully, Integration Server considers the transaction to be resolved and removes it from the list of unresolved XA transactions.
