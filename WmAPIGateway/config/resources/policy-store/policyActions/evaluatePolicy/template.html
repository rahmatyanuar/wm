<#list keys as key> <#if key=='apiKey'>
Require API Key Check
The API requires the usage of API Keys for Authentication and Authorization. There are 2 variants of passing along the API Keys.
API Key as HTTP Header Pass along the API key as ${apiKeyHeader}
Details ${apiKeyHeader}:api-key-value
Example ${apiKeyHeader}:fsffsfsd-fsfsdfsd-ewwerwe-fsdsad
API Key as Query Param Pass along the API key as URL query parameter - APIKey
Details APIKey=api-key-value
Example http://{host_name}:{port}/ws/SomeAPI/resource?APIKey=fsffsfsd-fsfsdfsd-ewwerwe-fsdsad
<#elseif key=='oAuth2Token'>
Evaluate OAuth2 Token
The API requires OAuth2 token authentication to allow access. To obtain access the following steps must be taken:
1. Register to consume the API. A email will be sent once the consumption has been approved.
2. The email will contain details how to create a OAuth2 token.
3. After creation of the OAuth2 token, the token must be provided in the header of each API call. The header will look like:
Details Authorization: Bearer {OAuth2 token}
Example Authorization: Bearer 0a9563286896432
<#elseif key=='jwtClaims'>
Evaluate JWT
The API requires JWT authentication token to allow access. Obtain a JWT token from the JWT Token provider and the token must be provided in the header of each API call. The header will look like:
Details Authorization: Bearer {JWT token}
Example Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJrZXkiOiJ2YWwiLCJpYXQiOjE0MjI2MDU0NDV9.eUiabuiKv-8PYk2AkGY4Fb5KMZeorYBLw261JPQD5lM
<#elseif key=='openIdClaims'>
Evaluate OpenId Connect
The API requires OpenId Connect authentication to allow access.
<#elseif key=='httpsCertificate'>
Evaluate Client Certificate for SSL connectivity
The API requires the usage of SSL with client certificates to allow access. When using this API, the API provider will provide details about the certificate usage.
<#elseif key=='httpBasicAuth'>
Evaluate HTTP Basic Authentication
The API requires the usage of HTTP Basic authentication header to allow access. The HTTP header should be passed in the following format:
Details Authorization: Basic {Base64 hash of username:password}
Example Authorization: Basic U29mdHdhcmVhZzpTb2Z0d2FyZWFn
<#elseif key=='ipAddressRange'>
Evaluate IP Address
The API requires that the IP address of the client matches against a trusted set. When using this API, please provide the API provider with details about the clients that will be accessing the API such that they can be added to the trusted set.
<#elseif key=='payloadElement'>
Evaluate XPath ExpressionThe API requires the usage of a element or attribute within the SOAP header or message to verify or route the message appropriately. When using this API, the API provider will give details on how the SOAP header or message will be interpreted.
<#elseif key=='hostNameAddress'>
Evaluate Hostname
The API requires that the hostname of the client matches against a trusted set. The HTTP header HOSTNAME is used to verify the hostname against the trusted set. When using this API, please provide the API provider with details about the clients that will be accessing the API such that they can be added to the trusted set.
